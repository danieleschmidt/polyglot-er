### Three regimes, empirically separated

- **Same-script** (en_de, en_fr): T4 80–83%, T3marg 4–6%, combined 83–87% F1.

- **Logographic** (en_zh, en_ja after pykakasi): T4 31–35%, T3marg 7–22%, combined 41–53%.

### Structural finding across non-Latin scripts

`Malmö` appears in the T3-only set (T4 misses, T3 catches) for **three independent non-Latin scripts**:
- en_ru: `Малмë` (T4 cosine 0.316)
- en_ar: `مالمو` (T4 cosine 0.282)
- en_ko: `말뫼` (T4 cosine 0.346)

Same proper noun, three scripts, same multilingual-MiniLM failure mode, same phonetic-T3 recovery. **This is the cleanest qualitative artifact for the §7 write-up.**

### Operational gotchas

- `pypinyin` + `pykakasi` are required `install_requires`; the venv must include both for the cascade to perform correctly on en_zh and en_ja. Without them, the fallback is silent and degrades T3 by 7–22pp on those pairs.
- The transliterator emits Hangul without internal whitespace; emitting spaces (the v1 behavior) costs +16pp T3 recall on en_ko due to Jaro-Winkler boundary penalties. Documented in `tests/test_normalization.py::test_transliterate_hangul_no_internal_spaces`.
- All evaluation scripts default to sentence-transformers (`paraphrase-multilingual-MiniLM-L12-v2`) when available. The TF-IDF fallback inflates T3 marginal by ~3× and is not paper-quotable.
- SPARQL extracts cache to `data/wikidata_raw/{lang}_{lang}/sitelinks.jsonl` (gitignored). Pilot reports and precision-eval JSON are checked in.

## Replication

Reproducing the final 7-pair F1 numbers from a clean checkout:

```bash
git clone https://github.com/danieleschmidt/polyglot-er
cd polyglot-er
python3 -m venv .venv && source .venv/bin/activate
pip install -e .[embeddings]

# Re-extract all 7 pairs at n=500/bucket (~30 min SPARQL traffic against
# query.wikidata.org; respects 5 QPS rate limit with retry/backoff).
python scripts/cross_pair_sweep.py \
    --pair en de --pair en fr --pair en ru --pair en ar \
    --pair en zh --pair en ja --pair en ko \
    --n-per-type 500

# Run precision-recall eval against hard negatives (~30 min CPU).
python scripts/precision_eval.py

# Run per-bucket F1 (~30 min CPU).
python scripts/per_bucket_f1.py
```

Final summaries land in `data/wikidata_raw/{cross_pair,precision,per_bucket}_summary.json`.
